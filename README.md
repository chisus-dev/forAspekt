## 1. `ansible задача.docx`

### Ansible

**Задача:** необходимо подготовить 3 типовых сервера. При знании Swarm, добавить ansible-код для введения серверов в кластер Swarm с labels `worker`.

**Требования к серверам:**

1. Использование пароля для входа по SSH — запрещено.
    
2. Заходить под пользователем `root` — запрещено.
    
3. На серверах должны быть учётные записи администраторов + технические.  
    Для примера добавляем:
    
    - `admin-1`
        
    - `admin-2`
        
    - `admin-3`
        
    - `ansible`
        
    - `autodeploy`
        
4. Должен быть установлен Docker фиксированной версии, какая есть в стандартном репозитории Ubuntu.
    
5. Необходимый софт:
    
    - `mc`
        
    - `ncdu`
        
    - `cifs-utils`
        
    - `nfs-common`
        

**Требования к технической реализации:**

1. Использование Ansible.
    
2. Использование Git.
    

Никаких ограничений по разработке Ansible нет, требуется сделать так, как считаете нужным.

## 2. `docker задача.docx`

### Docker

**Задача:** необходимо дать рекомендации/правки по улучшению/исправлению следующего Dockerfile:

```dockerfile
# Используем базовый образ

FROM python

# Указываем рабочую директорию
WORKDIR /app

# Копируем файлы проекта
COPY . .

# Устанавливаем зависимости 
RUN pip install -r requirements.txt

# Экспонируем порт для Flask
EXPOSE 5000

# Указываем команду запуска приложения
CMD ["python", "app.py"]
```

## 3. `swarm compose задача.docx`

### Задача

Необходимо подготовить `docker compose` файл для описания стека в Docker Swarm с сервисом, который включает docker-контейнер с образом:

```text
ubuntu:22.04
```

Необходимо реализовать связь с сервисом базы данных PostgreSQL и Redis.

У них есть сети:

```text
db-postgres-net
ds-redis-net
```

После чего необходимо подключиться к терминалу внутри контейнера в интерактивном режиме, установить туда необходимые пакеты и продемонстрировать возможность подключения к PostgreSQL и Redis.

### Требования к сервису

- Не менее 2 реплик.
    
- Контейнер должен работать постоянно, можно добавить команду:
    

```bash
sleep infinity
```

- Возможность бесшовного обновления — rolling update.
    
- Должна быть возможность просмотра логов на хост-сервере, нужно вынести volume.
    
- Лимит по ресурсам:
    
    - 1 ядро;
        
    - 500 МБ оперативной памяти.
        
- Лимит по логам:
    
    - 1 файл;
        
    - 5 мегабайт.
        
- Запущен на серверах с лейблом:
    

```text
SERVERTYPE = worker
```

- Внутри контейнера есть переменная окружения `HOSTNAME` с именем ноды, на которой запущен контейнер.
